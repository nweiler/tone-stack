# Meta-Patch: Next Steps & Objectives

This document tracks the prioritized features and technical goals for the Meta-Patch project.

## 1. Gear Catalog Sidebar (High Priority)
- **Goal**: Eliminate manual typing of gear names.
- **Task**: Implement a draggable sidebar in the Web Editor containing "Known Gear" (e.g., Tube Screamer, Plexi, Klon, 4x12 Greenback).
- **Benefit**: Ensures consistent naming across patches, which improves the accuracy of the platform adapters.

## 2. Real-Time Export Preview
- **Goal**: Instant validation of platform compatibility.
- **Task**: Add a toggle or third pane to the editor to view the generated **POD GO JSON**, **Bias FX JSON**, or **Amplitube XML** for the current visual design.
- **Benefit**: Allows power users to quickly "hack" their hardware presets using the Meta-Patch standard.

## 3. Library Search & Filtering
- **Goal**: Manage large collections of tones.
- **Task**: Add a monospace search bar to the `index` page to filter patches by name, tag (e.g., "High Gain"), or specific gear models.
- **Benefit**: Improves the "Tone Librarian" aspect of the tool.

## 4. Audio Integration (Phase 3)
- **Goal**: Make the patches "audible."
- **Task**: Create an adapter for **Neural Amp Modeler (NAM)** or **AIDA-X**.
- **Benefit**: Allows users to load a Meta-Patch and immediately hear a high-quality approximation using open-source neural profiles.

## 5. UI/UX Refinement
- **Task**: Add drag-and-drop animations.
- **Task**: Implement a "Trash Bin" for deleting components via drag.
- **Task**: Add visual "knob" or "slider" components to the settings editor for a more tactile feel.
